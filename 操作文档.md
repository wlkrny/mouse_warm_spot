## 操作流程

### 1. 打开视频
菜单栏 → 文件 → 打开视频 (Ctrl+O)

### 2. 绘制两个圈

| 圈 | 颜色 | 作用 | 如何画 |
|---|---|---|---|
| **ROI Core** | 绿实线 | 暖点铜片精确范围，判断是否压暖点 | 鼠标拖拽绘制 |
| **ROI Count** | 蓝虚线 | 两只鼠可能出现的范围，用于计数 | 绘制内圈后再拖拽一次 |

工具栏按钮可在编辑内圈/外圈间切换。

### 3. 标记参考帧 (0/1/2)

工具栏有 3 个按钮：

| 按钮 | 作用 |
|---|---|
| `[0] 标记背景` | 跳到空场帧，点击标记暖点无遮挡画面 |
| `[1] 标记1只` | 跳到只有1只鼠的帧，可多次点击追加样本 |
| `[2] 标记2只` | 跳到有2只鼠的帧，可多次点击追加样本 |

- **左键** = 追加样本
- **右键** = 撤回/清空
- **不要求按顺序**，[0] 未标时其他标记暂存，标了 [0] 后自动重算

### 4. 自动检测

点击工具栏 **"自动检测全视频"** → 进度条显示 → 完成后事件列表出现在左下角。

### 5. 审核事件

| 操作 | 快捷键 |
|---|---|
| 点击事件 | 视频跳转到该片段并循环播放 |
| 选择小鼠身份 | `1` `2` `3` `4` (多选 toggle) |
| 确认数量=1 | `Ctrl+1` |
| 确认数量=2 | `Ctrl+2` |
| 保存并下一个 | `Enter` |
| 标记误检 | `X` |
| 清空选择 | `C` |
| 播放/暂停 | `Space` |
| 单帧前进/后退 | `←` `→` |
| 快进/快退 | `D` `A` |
| 上/下一个事件 | `J` `L` |
| 刷新计数 | `R` |

事件列表颜色：
- 🟡 黄 = 待审核
- 🟢 绿 = 已确认
- 🔴 红 = 误检
- 🟠 橙 = 低置信需审核

### 6. 颜色识别 (耳标辅助)

点击工具栏 **"颜色识别"** 后选择 **“当前事件”**、**“全部事件”** 或取消。全部事件跳过 0 鼠片段，默认最多 **2** 个任务同时识别；设置 `MOUSE_COLOR_BATCH_CONCURRENCY=1` 可串行，最大值 4，非法值自动回退为 2。每任务独立读取视频，不增加单个 segment 的 VLM 请求预算；并发会提高 provider 同时请求/成本峰值。进度显示真实完成数/总数和事件 ID；取消后不再启动新任务，已运行/完成结果保留，失败事件不会阻塞其余事件。批处理结束显示总数、成功、测温器干扰、需复核、失败/取消和映射摘要。

### 7. 导出

| 菜单 | 快捷键 | 格式 |
|---|---|---|
| 文件 → 导出CSV | Ctrl+E | 30字段 CSV (UTF-8 BOM) |
| 文件 → 导出Markdown统计表 | - | 完整时间线 + 小鼠汇总 + 校准摘要 |

## 核心算法摘要

### 占据检测 (ROI Core 绿圈内)
- 粗筛：每10帧检测 ROI Count 蓝圈区域；外圈只负责发现候选，不能决定 clip 是否持续。
- 精查：触发后回溯2秒逐帧检测，并且**只以 ROI Core 的占据连续性**建立 OccupancyEpisode。
- Core 连续空闲达到 `core_gap_tolerance_seconds`（默认 0.3 秒）即结束并切开 clip；空闲为 `is_occupied=false` **或**综合遮挡率 `occlusion_area_ratio < core_empty_occupancy_threshold`（默认 0.04，严格 clamp 为 0~1）。低遮挡率优先于过时 occupied 标记；0.04 明显低于 0.08 release，避免正常噪声误切。外圈有鼠但 Core 为空时同样按此规则切开。
- 两个已确认占领 clip 间满足阈值的空档会成为独立 `estimated_mouse_count=0` 的 CountSegment（`start_reason=core_empty_gap`），记录 `core_empty_reason` 和 occupancy min/max/mean/threshold，GUI/导出可排错；视频前后纯背景不会生成此类行。
- 检测日志会显示该 Core-gap 参数；计数子片段严格在已切开的 episode 内生成，不会跨越 Core 空档重新合并。

### 数量估计 (ROI Count 蓝圈内)
- 前景 = 强暗色(V<55) OR (暗候选(V<95) AND 背景差异>25)
- 连通区先按 Core 连通性过滤：只保留直接重叠（≥20px 且≥Core 2%）或明确 3px 容差内接触 ROI Core 的前景块；外圈独立鼠不参与面积或 blob 计数。之后才碎片合并并计数。
- Debug 字段 `core_connected_blob_count`、`ignored_outer_blob_count`、`core_connected_area` 用于排查参与/排除的块。
- 面积计数：面积比 <1.7→1只, 1.7-2.7→2只
- 单blob最多判2只低置信，绝不判3/4

### 耳标颜色
- 搜索 ROI Count 环形区域(排除ROI Core)
- 三轮帧扫描
- 条件: V>80 S>30 H非暖色, 3-500px, 靠近小鼠身体≤20px
- 分类优先级: GPT AI视觉上下文模型 → ONNX CNN → HSV规则
- VLM 上下文：每 segment 仅 1 次请求，携带 9 帧大范围上下文图像（开头/中间/结尾各 3 帧，帧间隔 5）；提示词只准按实际占领/贴近或进入 ROI Core 的暖点鼠判断鼠数和耳标，外圈/角落鼠必须忽略。
- 上下文 crop：以 ROI 核心为中心，搜索 ROI (Core×2.8) × 1.5 的矩形区域；实际发送和 debug 保存的图像都用细青色椭圆定位 Core，避免遮挡耳标
- 短 segment：帧数不足时去重，安全 clamp 到 [start, end]
- VLM 结果融合：合法且置信度≥0.3 的 VLM 会重新确认该 segment 同时占据暖点的是 1 只或 2 只，并优先同步本地计数及有序颜色；AI 失败、unknown、低置信度或无效 JSON 时完整回退 CNN/HSV 与本地计数
- 测温器干扰：同一 9 帧 VLM 请求还返回严格 boolean `thermometer_present`，用于判断手持测温器/温度探头等非鼠设备是否遮挡或侵入暖点（不会从 reasoning 或自然语言推断）。缺字段兼容为 false，非 boolean 整条结果无效。仅在 VLM 已接受且明确 true 时，保留本地 count/colors、将 `identity_confidence=0.0`、`identity_needs_review=true`、`identity_conflict=false`，并在 `identity_method` 添加 `thermometer_detected`，提示“检测到测温器/探头，颜色识别置信度已置零，请人工复核”
- 双鼠第二色为 unknown 或颜色重复时标记人工复核，不凭规则虚构第二色；人工 confirmed segment 不会被自动覆盖
- 完整、无冲突的已知颜色会自动映射为鼠号：首次发现颜色分配最小可用 1–4（green 首次→1，red 后续→2），同色稳定复用；映射位于 `~/.mouse_warm_spot/color_mouse_mapping.json`，不保存密钥。0 鼠、unknown、重复/冲突和测温器结果绝不自动分配。使用 **视图 → 管理颜色-鼠号映射...** 查看或重置全部映射。

### AI 视觉模型配置（可选）

系统默认使用 HSV 规则进行耳标颜色分类。
启用 AI 视觉后，每个 segment 发送一次多帧上下文请求（而非每轮廓分别请求），
以控制成本。

**上下文帧选择**：
- 每 segment 最多选 9 帧：头三帧 (start, start+5, start+10)、中间三帧 (mid-5, mid, mid+5, mid=(start+end)//2)、尾三帧 (end-10, end-5, end)
- 足够长且不重叠的 segment 返回 9 帧；短 segment 因边界 clamp/去重可少于 9
- 每帧裁剪大范围外部图像：以 ROI core 中心为中心，短轴/长轴 = 搜索 ROI × 1.5
- 搜索 ROI = Core ROI × SEARCH_SCALE (2.8)

需配置以下环境变量（全部必填，无默认值）：

```bash
# 直接注入进程环境或通过启动器配置，不要依赖 .env 自动加载
MOUSE_COLOR_AI_PROVIDER=gpt           # gpt / hsv（本功能仅使用 GPT）
GPT_API_KEY=<your-api-key>            # GPT API 密钥
GPT_API_BASE=<provider-base-url>      # 从供应商文档获取
GPT_VISION_MODEL=<vision-capable-model-id>  # 支持视觉的模型名

# 每个 segment 消耗 1 次 VLM 预算（0=禁用 AI 视觉）
MOUSE_COLOR_AI_MAX_REQUESTS_PER_SEGMENT=3
```

注意：项目不包含 python-dotenv，创建 `.env` 文件不会自动生效。
请将环境变量导出到进程环境中（`export KEY=VAL`）或由用户启动器注入。
若用户的启动器支持显式加载 `.env`，可将上述内容写入 `.env` 作为可选方案。

环境变量概览：

| 环境变量 | 说明 |
|---|---|
| `MOUSE_COLOR_AI_PROVIDER` | `gpt` / `hsv`（默认 hsv；本功能仅使用 GPT） |
| `KIMI_API_KEY` | Kimi API 密钥 |
| `KIMI_API_BASE` | Kimi API 端点 URL（必填，不可猜测） |
| `KIMI_VISION_MODEL` | Kimi 视觉模型名（必填，不可猜测） |
| `MINIMAX_API_KEY` | MiniMax API 密钥 |
| `MINIMAX_API_BASE` | MiniMax API 端点 URL（必填，不可猜测） |
| `MINIMAX_VISION_MODEL` | MiniMax 视觉模型名（必填，不可猜测） |
| `OPENROUTER_API_KEY` | OpenRouter API 密钥 |
| `OPENROUTER_API_BASE` | OpenRouter API 端点 URL（必填，不可猜测） |
| `OPENROUTER_VISION_MODEL` | OpenRouter 视觉模型名（必填，须支持视觉输入） |
| `MOUSE_COLOR_AI_TIMEOUT` | 请求超时秒数（默认 30） |
| `MOUSE_COLOR_AI_MAX_TOKENS` | AI 视觉输出 token 上限，默认 512。对带 reasoning 的模型（如 MiniMax M3）建议 ≥512；若日志报告截断可进一步提高 |
| `MOUSE_COLOR_AI_MAX_REQUESTS_PER_SEGMENT` | 每段最大 VLM 请求数（默认 3，0=禁用。每段仅消耗 1 次） |
| `MOUSE_COLOR_AI_SAVE_DEBUG` | `1`=保存 VLM 上下文 crops/response/manifest 到 debug_vision_output/ |

视觉模型识别成功时，`identity_method` 字段为 `ear_tag_color_vlm`；
API 异常时自动回退到 CNN 或 HSV 规则（消耗该次预算），不中断检测。

**Multi-frame context API**：
- 新增 `classify_frames(list[np.ndarray])` 多图上下文 API（向下兼容单图 `classify(patch_bgr)`）
- 多图 payload 在同一 user message 中按时间顺序排列
- 提示词明确说明这是同一 clip 的前/中/后上下文帧
- 严格响应契约：`{"mouse_count": 1|2, "colors": ["red|yellow|blue|green|white|unknown"], "confidence": 0.0-1.0}`；colors 长度必须等于 count。只接受最终 content 的完整 JSON，不读取 reasoning；旧 `{color, confidence}` 响应兼容为单鼠。
